# Security Policy

This repository documents system governance and operational rules.
It must never contain sensitive or identifying information.

---

## Prohibited Content

The following must **never** be committed:

- Patient data (any form)
- Real doctor or staff names
- Emails, phone numbers, IDs
- Hospital, ministry, or department identifiers
- Screenshots from production
- Database exports from live systems

All examples must be fully anonymized.

---

## Data Sensitivity Boundary

This repository is intentionally limited to:
- Architecture
- Rules
- Governance logic
- Operational procedures

Any deployment data belongs exclusively in the production environment.

---

## Vulnerability Reporting

If a security risk or accidental exposure is discovered:
- Do **not** open a public issue with details
- Remove the content immediately if possible
- Escalate privately to the repository owner

---

## Scope Disclaimer

This repository does not replace:
- Hospital data governance policies
- National data protection regulations
- Internal IT security controls

It is a **documentation and specification artifact**, not a data store.
